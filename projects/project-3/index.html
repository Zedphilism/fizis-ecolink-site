1.0 Introduction

This submission focuses on translating user needs and task requirements into tangible interface designs for the EcoLink mobile application. The work is presented in two parts: conceptual design and physical design. The conceptual stage defines system structure and task flow. The physical stage converts those decisions into concrete screens and interactions.

3.0 Conceptual Design

3.1 System Concept and Core Functions

EcoLink is a mobile application that supports waste management and community services by combining information discovery, civic reporting, and service booking. The core modules include authentication, dashboard updates, locating recycling or disposal locations, service booking such as RORO bin booking, gig worker hiring and job posting, complaint reporting, and profile management.

Authentication
Login, register, account access
Find
Recycling centres, disposal sites
Services
RORO booking, gig work hire and listing

3.2 Initial Wireframe (Low Fidelity)

The initial wireframe focused on capturing the fundamental screens and features needed for EcoLink without prioritizing refined flow. It was used to confirm screen coverage and information placement before deeper logic refinement.

  • Login and register
  • Dashboard announcements and updates
  • Find location feature
  • Service page with RORO and gig functions
  • Report issue and profile

3.3 Refined Wireframe

The refined wireframe improved navigation flow and task structure, especially for RORO booking and gig related activities. Functions were grouped more logically and steps were clarified to reduce friction during task completion.

  • Clearer sequence for RORO booking steps
  • Better separation of Find, Service, and Report
  • More consistent structure across screens

4.0 Physical Design

4.1 From Wireframe to Prototype

The physical design stage translates the conceptual structure into concrete screen layouts. This includes component placement, visual hierarchy, spacing, and interaction elements suitable for mobile. The goal is consistent and clear navigation while reducing user effort during key tasks.

4.2 First Prototype (Iteration 1)

Open

This version focuses on validating layout and navigation. It was used to identify early usability issues and refine interaction flow.

  • Initial component consistency
  • Early spacing and hierarchy decisions
  • Basic task flow validation

4.3 Final Prototype (Iteration 2)

Open

The final prototype refines clarity, improves visual consistency, and strengthens navigation across the application. This is the most complete version of the intended user experience for EcoLink.

  • Improved readability and information hierarchy
  • More consistent UI patterns across screens
  • Smoother flow for booking and reporting tasks

5.0 Key Screen Breakdown

Dashboard

The main entry point for announcements, updates, and quick access to core functions.

  • Highlights what is new
  • Supports quick access to key tasks

Find (Recycling and Disposal)

Helps users locate recycling centres or disposal sites based on location and relevant inputs.

  • Map based discovery
  • Clear location guidance and inputs

Book RORO Bin

Booking flow supports selection, scheduling, provider comparison, and payment confirmation.

  • Step based progression
  • Cost transparency before payment

Report Issue

Enables users to submit issue details, location, and attachments for evidence.

  • Structured reporting to reduce ambiguity
  • Attachment support for proof

6.0 Design Justification

The interface design emphasizes consistency, clear grouping, and minimal user effort. Functions are organized by intent, such as discovery (Find), action (Services), and reporting (Report). Visual hierarchy and spacing guide attention and improve readability on mobile devices. Interaction patterns follow standard mobile conventions to reduce learning time.

7.0 Conclusion

This submission demonstrates the transition from conceptual planning to physical interface design through iterative refinement. By producing an initial wireframe, refining task flow, and developing high fidelity prototypes, the EcoLink design is structured to support clear and efficient user interaction. This provides a solid base for evaluation and usability testing in the next phase.
