Project 02 - Establishing Requirements

Phase: User and task analysis

This page documents the user analysis and task analysis used to establish requirements for EcoLink, a proposed smart waste management system. Three personas were defined, and three real users performed think aloud testing on existing platforms to uncover usability issues and translate them into clear functional and non-functional requirements.

1. Introduction 2. User analysis 3. Task analysis 4. Refined HTA 5. Established requirements

1. Introduction

This stage establishes requirements for EcoLink by analysing user characteristics, goals, and needs, then validating task flows through think aloud testing on existing municipal style services. The focus is to identify task friction, usability problems, and improvement opportunities, then turn those findings into requirements that guide the next design phase.

Proposed tasks

  • Task 1: Lodge a waste-related complaint
  • Task 2: Locate waste disposal and recycling sites (including e-waste centres)
  • Task 3: Book a RoRo bin for bulk waste collection

2. User analysis

Three personas represent key user groups for EcoLink: a young housewife, a private contractor manager, and an elderly gardening enthusiast. Each persona is paired with a realistic scenario aligned to the three proposed tasks.

2.1 Personas

Aina Binti Rahman

Age: 28
Occupation: Full-time housewife
Technology experience: Moderate mobile application user

Aina lives at Taman Budi, Kuala Lumpur. When bins overflow, she worries about hygiene, pests, and her children’s safety. She dislikes calling council offices because long holds and confusing instructions feel stressful. She prefers a mobile app that solves the problem quickly with minimal steps.

Primary goals
  • Quickly report issues like missed collection.
  • Receive instant confirmation about her report.
Secondary goals
  • Avoid stress and time wasted through clear tracked communication.
  • Restore a hygienic environment as soon as possible.
Pains
  • Missed collection causes an inconvenient and risky environment.
  • Unclear estimated resolution timeline.
  • Confusing report categories may send the report to the wrong department.
Gains
  • Effortless reporting with photo and location in under a minute.
  • Trackable progress and status visibility.
  • Clear confirmation and estimated resolution time.
“Saya nak lapor masalah cepat, senang, dan tahu bila orang datang bersihkan.”

Amir Bin Fauzi

Age: 40
Occupation: Construction Project Manager
Technology experience: High, daily professional digital user

Amir manages waste disposal for renovation and construction sites. Debris must be cleared quickly to avoid hazards, delays, and potential fines. He prefers efficient, clear, reliable solutions that reduce calls, paperwork, and miscommunication.

Primary goals
  • Book RoRo bins quickly without calls or paperwork.
  • Track service status clearly: availability, delivery, pickup.
Secondary goals
  • Access transparent pricing and billing details.
  • Maintain documentation for compliance and site audits.
Pains
  • Slow or unresponsive waste disposal services.
  • No real time bin availability and unclear delivery timing.
  • Manual paperwork introduces errors and wastes time.
  • Repeated calls just to confirm status.
Gains
  • Live availability and delivery slots.
  • Instant confirmations and digital receipts shareable with site teams.
  • More reliable flow with less miscommunication.
  • Faster workflow with less administrative hassle.
“If I can see availability, price, and delivery status in one place, I can keep the site running on schedule.”

Hassan Bin Omar

Age: 62
Occupation: Retired school teacher
Technology experience: Basic smartphone user

Pak Hassan spends most of his time gardening. He wants to dispose of garden waste, e-waste, and special items responsibly, but often feels unsure where different waste types should go. He struggles with small text and complicated menus and prefers simple instructions and easy navigation.

Primary goals
  • Find the nearest correct disposal or recycling location.
  • Get clear step by step directions without complicated menus.
Secondary goals
  • Understand what type of waste goes where.
  • Avoid wasted trips to the wrong location.
Pains
  • Small text and too many buttons.
  • Uncertainty about special waste handling.
  • Frustration when he reaches the wrong site.
Gains
  • Clean simple interface with large fonts.
  • Automatic suggestions for disposal categories.
“Saya cuma nak aplikasi yang jelas, tulisan besar, dan tunjuk jalan ke tempat buang yang betul.”

2.2 Scenarios

Each persona is paired with a scenario that describes how they would use EcoLink to complete one of the three tasks.

Persona: Aina Binti Rahman Task 1: Lodge complaint

Scenario 1: Lodge a Waste Complaint

Aina notices rubbish bins near the playground have not been collected for two days. The smell is unpleasant and stray animals gather. She opens EcoLink, selects Lodge Complaint, takes a photo, GPS tags the location, chooses a category, and enters a short description. After submission, she receives a reference number and a notification confirming the report is forwarded to the local team. She expects to track the complaint status without needing to call or visit a counter.

Persona: Hassan Bin Omar Task 2: Locate disposal sites

Scenario 2: Locate Waste Disposal Sites

Pak Hassan needs to dispose of unwanted items including e-waste. He opens EcoLink and taps Locate Disposal Sites. Using filters, he selects E-waste and enters his residential area. The system displays a list of nearby authorised centres with distance info. He selects the closest location and uses the map for step by step navigation with large text and simple visuals.

Persona: Amir Bin Fauzi Task 3: Book RoRo bin

Scenario 3: Book a RoRo Bin

Amir requires a RoRo bin to clear construction waste at a renovation site. He accesses EcoLink via the web portal, selects Book RoRo Bin, enters the site location, chooses a bin size, and selects a delivery date. The system shows availability and pricing immediately. Amir confirms and receives a digital confirmation and tracking details.

Findings (discussion)

  • All three users completed submission, but difficulty differed across users.
  • User 1 hesitated during location entry steps.
  • User 2 moved quickly but was frustrated by unnecessary prompts and confirmation steps.
  • User 3 struggled to locate the complaint page and position the map accurately, requiring assistance.
  • Complaints were faster when location services were automatically activated.
Requirement specification - Functional
  • The system shall allow users to upload photo evidence during complaint submission.
  • The system shall automatically detect and pre-fill location information using GPS.
Requirement specification - Non-functional
  • The system shall minimise redundant confirmation prompts after submission.
  • The interface shall reduce manual data entry wherever possible.

Findings (discussion)

  • Users reported mild confusion when selecting waste categories and unclear terminology.
  • User 1 relied on distance and visual ratings, User 2 prioritised search fields, User 3 scrolled manually rather than using filters.
  • Map clutter and small icons made navigation harder, especially for elderly users.
  • Behaviours suggest a need for both structured filters and simplified browsing modes.
Requirement specification - Functional
  • The system shall support both keyword search and categorical filtering.
  • The system shall provide clear waste type categories.
Requirement specification - Non-functional
  • The map interface shall use large high contrast icons and simplified navigation routes.
  • The system shall prioritise accessibility for elderly users.

4. Refined HTA (improved EcoLink flow)

Based on usability issues found during task analysis, the refined HTA below streamlines each task using quick actions, fewer steps, less manual input, and clearer guidance for eligibility and bin size selection.

Refined HTA - Task 1: Lodge a Complaint
Task Goal: 0.0 Lodge a Complaint
  1. Launch the application
    1. Locate the application on the mobile device
    2. Tap to open the application
    3. Wait for the home screen to load
  2. Access quick action for reporting
    1. Tap the Quick Action: Report button
  3. Enter complaint details on a single page
    1. Select issue type
    2. Select urgency level
    3. Set complaint location
    4. Enter description of the issue
    5. Take or upload photo evidence (optional)
  4. Submit the complaint
    1. Review entered information
    2. Tap submit button
    3. Wait for submission confirmation
Plan 0 (Main Plan): Do 1 → 2 → 3 → 4 in sequence.
Design rationale: supports photo evidence upload, reduces manual entry through GPS, consolidates input on one page for easier review and correction.
Refined HTA - Task 2: Locate Waste Disposal Site
Task Goal: 0.0 Locate a Waste Disposal or Recycling Site
  1. Launch the application

    3. Wait for the home screen to load
  2. Access recycling quick action

  3. Search for waste disposal sites
    1. Choose search by location

    3. Apply filters based on user needs
    4. View list of available waste disposal sites

Design rationale: supports waste type filter categories and improves map accessibility with larger icons, reduced clutter, and simpler navigation.

Design rationale: adds bin size recommendation with visual guidance and makes eligibility clear early, reducing abandonment and improving confidence.

5. Established requirements

The consolidated requirements below are derived from the task analysis findings and are proposed as the key design requirements for EcoLink.

5.1 Functional requirements

  • The system must allow users to upload or capture photo evidence when lodging complaints.
  • The system must provide waste type filter categories (for example domestic waste, recyclables, e-waste, chemicals).
  • The system must include a RoRo bin size recommendation tool with visual guidance.

5.2 Non-functional requirements

  • The system must support automatic GPS location detection for complaint submissions.
  • The system must provide an accessible map interface with enlarged icons and simplified navigation.
  • The system must display a task eligibility indicator (household vs commercial users) before RoRo booking begins.
