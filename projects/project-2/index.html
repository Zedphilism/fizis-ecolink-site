Project 02 – Establishing Requirements

Phase: User & task analysis

This page documents the user and task analysis for EcoLink, a smart waste management concept inspired by existing local council complaint portals such as MBIP/MBPJ. We focus on three core tasks – locating recycling bins, filing complaints, and booking RoRo bins – to understand real user needs and translate them into design requirements.

1. Introduction 2. User analysis 3. Task analysis 4. Refined HTA 5. Final requirements

1. Introduction

For this stage, we analyse how users would interact with EcoLink while performing day-to-day waste-related tasks. EcoLink is evaluated as a future replacement / improvement of existing local authority systems (for example MBIP/MBPJ complaint and waste portals). The goal is to discover usability issues, capture user expectations, and shape clear design requirements.

Chosen tasks

  • Task 1: Locate recycling bin / disposal site.
  • Task 2: File a waste-related complaint.
  • Task 3: Book a RoRo bin for bulky or construction waste.

2. User analysis

Three personas were developed to represent key user groups: a residential housewife, a construction project manager, and an elderly gardening enthusiast. Each persona is paired with a realistic scenario that reflects how they interact with EcoLink.

2.2 Scenarios

Each persona is paired with a realistic scenario that illustrates how they would use EcoLink to complete one of the three core tasks.

Persona: Aina Rahman Resident Task 2 – File complaint

Scenario 1 – Lodge a Waste Complaint

Aina notices that rubbish bins near the children's playground have not been collected for two days. The smell is unpleasant and stray animals start to gather. Concerned about hygiene and her children’s safety, she opens the EcoLink mobile app and selects the “Lodge Complaint” feature.

  1. She takes a photo of the overflowing bins using the in-app camera.
  2. The app automatically tags her location via GPS.
  3. She selects the appropriate complaint category and writes a short description.
  4. Upon submission, the system generates a reference number and confirmation screen.
  5. A notification indicates that the report has been forwarded to the municipal cleaning team.
Persona: Hassan Omar Elderly gardening enthusiast Task 1 – Locate recycling bin

Scenario 2 – Locate Waste Disposal Site

After clearing old gardening tools and damaged electrical equipment from his shed, Pak Hassan needs to dispose of several items responsibly. Unsure where to bring e-waste, he opens the EcoLink app and taps “Locate Disposal Sites”.

  1. He selects “E-waste” as the waste category.
  2. The system shows nearby authorised recycling centres with distance estimates.
  3. He chooses the closest site and opens the map view.
  4. Large text labels and simple icons guide him step by step.
  5. He travels to the centre confident he is disposing of e-waste correctly.
Persona: Amir Fauzi Construction project manager Task 3 – Book RoRo bin

Scenario 3 – Book a RoRo Bin

Amir is preparing for debris removal at an active renovation site and requires a RoRo bin to clear waste efficiently. He accesses EcoLink via the web portal and chooses the “Book RoRo Bin” service.

  1. He enters the project location and selects the required bin size.

  3. He confirms the preferred delivery window and reviews the summary.
  4. After submitting, he receives a digital confirmation with reference ID.
  5. Delivery status can be tracked from the portal so he can plan site work safely.

Hierarchical Task Analysis (current system)

Task 1 – HTA for User 1
  1. Open EcoLink / council portal.
  2. Navigate to “Locate disposal sites”.
  3. Select waste category (e.g., recycling, e-waste).
  4. Allow or enter location.
  5. Browse list/map of nearby locations.
  6. Tap one location to view details (address, opening hours).
Task 1 – HTA for User 2
  1. Open EcoLink homepage.
  2. Use search/filter to choose waste type.
  3. Zoom and pan map to area of interest.
  4. Compare distance of several sites.
  5. Choose preferred site and save it / start navigation.
Task 1 – HTA for User 3
  1. Open EcoLink app on phone.
  2. Tap “Nearby bins & centres”.
  3. Scroll through a simplified list with distance.
  4. Tap a card to open map view.
  5. Follow turn-by-turn directions.

Findings (discussion)

  • Some users struggled to notice filters for waste type and initially saw irrelevant locations.
  • Older users like Hassan needed larger text and cleaner map labels to feel confident.
  • Think-aloud sessions showed confusion when the system mixed public recycling bins with private recyclers.
Design requirements – Functional
  • System must allow filtering by waste category (recycling, e-waste, garden waste, etc.).
  • System must use GPS to suggest nearby sites within a configurable radius.
  • Each site must display key details: address, opening hours, accepted waste types.
Design requirements – Non-functional
  • Map and list views must remain readable for elderly users (font size, contrast).
  • Search results should appear within a few seconds after filter changes.
  • Information must be accurate and updated regularly to avoid wrong trips.

Hierarchical Task Analysis (current system)

Task 2 – HTA for User 1
  1. Open complaint portal / app.
  2. Choose complaint type (e.g., “Missed collection”).
  3. Fill in location details (address / GPS).
  4. Type description of the issue.
  5. Attach photo as evidence.
  6. Submit and receive complaint reference number.
Task 2 – HTA for User 2
  1. Log in to portal with account.
  2. Select “New complaint”.
  3. Pick category and subcategory.
  4. Use map to drop a pin on exact location.
  5. Attach multiple images if needed.
  6. Submit and check status page.
Task 2 – HTA for User 3
  1. Dial hotline number shown on website.
  2. Navigate IVR menu to reach relevant department.
  3. Provide complaint details verbally.
  4. Write down reference number (if given).
  5. Wait for follow-up message or call.

Findings (discussion)

  • Users became confused when category labels were too technical or overlapping.
  • Entering location manually slowed down the process and led to mistakes.
  • Think-aloud revealed anxiety about whether complaints would actually be actioned.
Design requirements – Functional
  • System must support photo and GPS tagging directly from the app.
  • Complaint categories must be simplified with clear examples.
  • Users must be able to track status (Received, In progress, Completed).
Design requirements – Non-functional
  • Confirmation screens and notifications must be immediate and unambiguous.
  • Form layout must minimise typing on mobile devices.
  • Language should avoid technical jargon so residents can understand easily.

  2. Select “Book RoRo Bin”.
  3. Enter project site location and choose bin size.
  4. Select delivery date and pickup date.
  5. Review cost breakdown and terms.
  6. Submit booking and receive digital confirmation.
Task 3 – HTA for User 3
  1. Log in to existing council system.
  2. Navigate through several menus to find bulky waste / RoRo service.
  3. Download offline form or PDF.
  4. Fill in manually and upload / email back.

Findings (discussion)

  • Existing processes often require phone calls or manual forms, which slow down urgent site work.
  • Users wanted clear pricing and visibility of bin availability before committing.
  • Think-aloud revealed uncertainty about whether bookings included permits or additional fees.
Design requirements – Functional
  • System must support end-to-end online RoRo booking (size, date, location).
  • Users must see live availability for selected dates.
  • System must generate a clear digital confirmation and reference ID.
Design requirements – Non-functional
  • Pricing and additional charges must be transparent and itemised.
  • Interface should support frequent use by professionals such as project managers.
  • System should be reliable enough for time-sensitive construction schedules.

4. Refined HTA (improved system)

Based on the issues discovered in the initial task analyses, the following refined HTAs describe how EcoLink should ideally support each task. These flows remove unnecessary steps, reduce cognitive load, and align with the functional and non-functional requirements.

Refined HTA – Task 1: Locate Recycling Bin
  1. Open EcoLink app.
  2. Tap “Nearby bins & centres”.
  3. Select waste category (default: Common recycling).
  4. System automatically detects location and displays closest sites on map and list.
  5. User taps one site to view details (accepted waste, opening hours, navigation option).
  6. User starts navigation or saves site as favourite.
Refined HTA – Task 2: File Complaint
  1. Open EcoLink and tap “Report an issue”.
  2. Take photo (or choose from gallery) of the problem.
  3. Confirm auto-detected location; adjust pin if needed.
  4. Select simple category with example icons.
  5. Optionally add short description.
  6. Submit and receive confirmation + reference number + estimated response time.
  7. Track status from “My reports” page.
Refined HTA – Task 3: Book RoRo Bin
  1. Log in to EcoLink portal as registered user.
  2. Go to “RoRo bin booking”.
  3. Enter site location or select from saved projects.
  4. Choose bin size and preferred delivery / pickup dates.
  5. Review automatically calculated price and permit requirements.
  6. Confirm booking and receive digital confirmation with reference ID.
  7. Monitor delivery status and manage changes from “My bookings”.

5. Final established requirements

Combining insights from personas, scenarios, task analysis, and refined HTAs, we derived the following core functional and non-functional requirements for EcoLink.

5.1 Functional requirements

Task 1 – Locate recycling bin / disposal site

  • System shall detect user location and display nearby disposal sites on map and list view.
  • System shall allow filtering by waste category (recycling, e-waste, garden waste, bulky waste).
  • System shall show key site information: address, opening hours, accepted materials.

Task 2 – File complaint

  • System shall let users submit complaints with photo, location, category, and optional description.
  • System shall generate a unique reference ID for every complaint submitted.
  • System shall provide a status tracking screen for all complaints created by the user.

Task 3 – Book RoRo bin

  • System shall allow users to request RoRo bins by specifying site location, bin size, and dates.
  • System shall display available time slots and pricing before confirmation.
  • System shall send a booking confirmation and store it in the user’s account.

5.2 Non-functional requirements

Usability & accessibility

  • Interface shall use clear language and avoid technical jargon where possible.
  • Text and icons shall be large enough for elderly users such as Hassan to read comfortably.
  • Key actions (Report, Locate, Book) shall be reachable within three taps from the home screen.

Performance & reliability

  • Search and map results shall load within a few seconds on typical mobile connections.
  • Submitted data (complaints, bookings) shall not be lost if network connection briefly drops.
  • System shall provide clear error messages and recovery options when actions fail.

Trust & transparency

  • Complaint and booking statuses shall be visible and updated as work progresses.
  • Pricing, fees, and terms shall be displayed before confirmation, especially for RoRo bookings.
  • Notifications shall inform users when complaints are resolved or bins are delivered / collected.
