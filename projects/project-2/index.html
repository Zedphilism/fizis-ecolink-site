Project 02 – Establishing Requirements

Phase: User and task analysis

This page documents the user analysis and task analysis for EcoLink, a smart waste management concept inspired by existing local authority complaint portals. We focus on three core tasks, locating recycling centres, lodging complaints, and booking RoRo bins. Evidence is supported by thinking aloud sessions, HTA breakdowns, and the final design requirements derived from the analysis.

1. Introduction 2. User analysis 3. Task analysis 4. Refined HTA 5. Final requirements

1. Introduction

In this stage, we study how users currently complete real world waste related tasks and where the friction happens. The aim is to identify usability issues, user expectations, and practical constraints, then convert those findings into clear functional and non functional requirements for EcoLink.

Chosen tasks

  • Task 1: Locate recycling centre or disposal site.
  • Task 2: Lodge a waste related complaint.
  • Task 3: Book a RoRo bin for bulky or construction waste.

Summary outcome

Across the analysis, we derived concrete design requirements that strongly influence the next prototype iteration: photo evidence support, waste type filtering, RoRo bin size guidance, automatic GPS detection, accessible map design, and clear eligibility indicators between household and commercial flows.

2. User analysis

Three personas were developed to represent key user groups: a residential housewife, a construction project manager, and an elderly gardening enthusiast. Each persona is paired with a realistic scenario that reflects how they interact with EcoLink.

2.2 Scenarios

Each persona is paired with a realistic scenario that illustrates how they would use EcoLink to complete one of the three core tasks.

Persona: Aina Rahman Resident Task 2

Scenario 1 – Lodge a Waste Complaint

Aina notices that rubbish bins near the children's playground have not been collected for two days. The smell is unpleasant and stray animals start to gather. Concerned about hygiene and her children’s safety, she opens EcoLink and selects the lodge complaint feature.

  1. She takes a photo of the overflowing bins as evidence.
  2. The app tags her location automatically using GPS.
  3. She selects the complaint category using simple labels and examples.
  4. She submits and receives a reference number with confirmation.
  5. She tracks the status later without needing to call anyone.
Persona: Hassan Omar Elderly user Task 1

Scenario 2 – Locate Disposal Site

After clearing old gardening tools and damaged electrical equipment from his shed, Pak Hassan needs to dispose of several items responsibly. Unsure where to bring e-waste, he opens EcoLink and taps locate disposal sites.

  1. He selects the waste category, for example e-waste.
  2. The system shows nearby authorised recycling centres.
  3. He chooses the closest site and opens map view.
  4. Large icons and simplified navigation help him feel confident.
  5. He reaches the correct location without guesswork.
Persona: Amir Fauzi Construction user Task 3

Scenario 3 – Book a RoRo Bin

Amir needs a RoRo bin to clear debris at a renovation site. He uses EcoLink to book the bin and expects a workflow that is fast, transparent, and suitable for commercial projects.

  1. He enters site location and selects commercial eligibility.
  2. He uses the size recommendation tool to pick the right bin size.
  3. He selects delivery and pickup dates.
  4. He reviews itemised pricing and confirms booking.
  5. He receives a digital confirmation and tracks delivery status.

3. Task analysis

For each of the three tasks, users performed the task while thinking aloud. The recordings and observations are used as evidence to establish requirements. The links below are the official recordings that support this analysis. User 3 is pending and will be updated once available.

3.1 Task 1 – Locate Recycling Centre

Users try to find a nearby recycling centre or disposal point for specific waste types using the EcoLink concept flow.

User videos

User 1 Locate recycling centre recording. User 2 Locate recycling centre recording. User 3 Link pending.

Hierarchical Task Analysis (observed)

Task 1 – HTA for User 1
  1. Open the system and access locate feature.
  2. Select waste category (recycling, e-waste, chemicals).
  3. Allow location detection.
  4. View nearby results on map and list.
  5. Open a location card to view details.
  6. Start navigation.
Task 1 – HTA for User 2
  1. Enter locate page.
  2. Use filters to narrow down waste type.
  3. Compare results by distance.
  4. Pick a recycling centre and open details.
  5. Confirm accepted waste type before going.
  6. Navigate to location.
Task 1 – HTA for User 3
  1. Open EcoLink app on phone.
  2. Tap nearby bins and centres.
  3. Scroll through a simplified list with distance.
  4. Tap a card to open map view.
  5. Follow directions.

Findings (discussion)

  • Users needed clear filter categories to avoid irrelevant results.
  • Elderly users required enlarged icons and simplified navigation on map view.
  • Accuracy matters because wrong trips reduce trust quickly.
Design requirements – Functional
  • System must provide waste type filter categories (domestic waste, recyclables, e-waste, chemicals).
  • System must display nearby centres with distance and key details.
  • Each site must show accepted waste types clearly.
Design requirements – Non-functional
  • Map interface must be accessible with enlarged icons and simplified navigation.
  • Results should load fast after filter changes.
  • Information should be kept accurate and updated regularly.

3.2 Task 2 – Lodge Complaint

Users report waste related issues such as missed collection, illegal dumping, or overflowing bins.

User videos

User 1 Lodge complaint recording. User 2 Lodge complaint recording. User 3 Link pending.

Hierarchical Task Analysis (observed)

Task 2 – HTA for User 1
  1. Open complaint feature.
  2. Select complaint type.
  3. Capture or upload photo evidence.
  4. Confirm location through GPS.
  5. Add short description.
  6. Submit and receive reference number.
Task 2 – HTA for User 2
  1. Enter complaint page.
  2. Select category and subcategory.
  3. Pin exact location if needed.
  4. Attach evidence images.
  5. Submit complaint.
  6. Check status page later.
Task 2 – HTA for User 3
  1. Dial hotline shown on website.
  2. Navigate menu to reach department.
  3. Provide complaint details verbally.
  4. Write down reference if given.
  5. Wait for follow up.

Findings (discussion)

  • Users wanted proof based reporting, especially photos, to increase confidence.
  • Automatic GPS detection is critical because manual address entry is slow and error prone.
  • Status transparency reduces anxiety about whether action will happen.
Design requirements – Functional
  • System must allow users to upload or capture photo evidence when lodging complaints.
  • Complaint categories must be simple with clear examples.
  • System must generate a reference number and allow tracking.
Design requirements – Non-functional
  • System must support automatic GPS location detection for complaint submissions.
  • Confirmation feedback must be immediate and unambiguous.
  • Language must avoid technical jargon for residents.

3.3 Task 3 – Book RoRo Bin

Users book a RoRo bin for bulky or construction waste, choosing bin size, location, and delivery time. User 2 in this task is the group member performing the think aloud session.

User videos

User 1 Book RoRo bin recording. User 2 (group member) Book RoRo bin recording. User 3 Link pending.

Hierarchical Task Analysis (observed)

Task 3 – HTA for User 1
  1. Search online for RoRo bin service provider.
  2. Use call or chat to request quotation.
  3. Discuss bin size, dates, and location manually.
  4. Confirm booking.
  5. Wait for invoice or confirmation message.
Task 3 – HTA for User 2 (group member)
  1. Open EcoLink portal and select Book RoRo Bin.
  2. Select eligibility type (household or commercial).
  3. Enter location and waste type summary.
  4. Use size recommendation tool with visual guidance.
  5. Select delivery date and pickup date.
  6. Review itemised pricing and confirm booking.
  7. Receive digital confirmation and reference ID.
Task 3 – HTA for User 3
  1. Log in to existing council system.
  2. Navigate menus to find bulky waste service.
  3. Download form or PDF.
  4. Fill in manually and upload or email.
  5. Wait for approval.

Findings (discussion)

  • Manual booking processes are slow and risky for time sensitive construction schedules.
  • Users want clear bin size guidance to avoid picking the wrong size.
  • Eligibility clarity matters because household and commercial flows have different rules.
Design requirements – Functional
  • System must include a RoRo bin size recommendation tool with visual guidance.
  • System must support end to end online booking (size, date, location).
  • System must generate digital confirmation and reference ID.
Design requirements – Non-functional
  • Pricing must be transparent and itemised before confirmation.
  • System must display a task eligibility indicator (household vs commercial users).
  • Workflow should be reliable for professional repeated use.

4. Refined HTA (improved system)

Based on issues discovered in the task analyses, the refined HTAs below describe how EcoLink should support each task. The focus is reducing steps, lowering cognitive load, improving accessibility, and increasing transparency.

Refined HTA – Task 1: Locate Recycling Centre
  1. Open EcoLink app.
  2. Tap locate centres.
  3. Select waste type filter category.
  4. System detects location automatically and shows nearby results.
  5. User switches between list and accessible map view with enlarged icons.
  6. User opens a centre card to view accepted waste types, hours, and navigation option.
Refined HTA – Task 2: Lodge Complaint
  1. Open EcoLink and tap report issue.
  2. Capture or upload photo evidence.
  3. Confirm GPS detected location or adjust pin.
  4. Select complaint category with simple labels and examples.
  5. Optional short description.
  6. Submit and receive confirmation with reference number.
  7. Track complaint status from my reports.
Refined HTA – Task 3: Book RoRo Bin
  1. Open RoRo booking.
  2. Select eligibility type (household or commercial).
  3. Enter site location or select saved project.
  4. Use size recommendation tool with visual guidance.
  5. Select delivery and pickup dates.
  6. Review itemised pricing and terms.
  7. Confirm booking and receive digital confirmation with reference ID.

5. Final established requirements

Combining findings from personas, scenarios, task analysis, and refined HTA, we established the requirements below.

5.1 Functional requirements

Locate recycling centre

  • System shall provide waste type filter categories (domestic waste, recyclables, e-waste, chemicals).
  • System shall display nearby centres by distance with key details and accepted waste types.

Lodge complaint

  • System shall allow users to capture or upload photo evidence when lodging complaints.
  • System shall generate a unique reference ID for every complaint.
  • System shall provide status tracking for complaints created by the user.

Book RoRo bin

  • System shall include a RoRo bin size recommendation tool with visual guidance.
  • System shall allow bookings by specifying location, bin size, and delivery and pickup dates.
  • System shall display itemised pricing before confirmation and store confirmation in the user account.

5.2 Non-functional requirements

Usability and accessibility

  • System shall provide an accessible map interface with enlarged icons and simplified navigation.
  • Text and icons shall be readable for elderly users.
  • Key actions shall be reachable quickly from the home screen.

Performance and reliability

  • System shall support automatic GPS location detection for complaint submissions.
  • Submitted data shall not be lost during temporary connection drops.
  • System shall provide clear error messages and recovery options.

Trust and transparency

  • System shall display a task eligibility indicator (household vs commercial users).
  • Complaint and booking statuses shall be visible and updated as progress happens.
  • Pricing and terms shall be shown before confirmation.
